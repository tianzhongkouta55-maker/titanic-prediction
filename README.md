- **データの前処理 (`map`関数)**: 
  機械学習モデルが計算を扱えるよう、文字データ（Sex）を数値に変換。リスト要素を一括処理する効率的な手法を習得しました。
- **f-string による出力**:

  - **`.predict` (予測)**: 学習したルールを新しいデータに適用する「アウトプット」の工程。

## Day 2: 過学習の特定と特徴量エンジニアリングによる本質的ルールの抽出（クリックで展開）

### 1. 実施内容：モデルの最適化と特徴量生成
*   **決定木モデルの構築と制御**: `max_depth` パラメータを用いた学習の制限。
*   **特徴量エンジニアリング**: 既存の変数（SibSp, Parch）を統合し、新変数 **`FamilySize`** を作成。
*   **データ補完の実装**: `.fillna()` を用い、統計的根拠に基づいた欠損値（Age/Fare）の補完。

### 2. 解決した課題：過学習の抑制とノイズの排除
*   **過学習の特定**: 訓練データ（98.17%）とテストデータ（79.33%）の大きな乖離を、モデルの「丸暗記」によるものと特定し、階層制限により解消。
*   **重要度スコアの健全化**: 運賃や年齢などの「細かすぎる数値データ」に判断が偏っていた状態を改善し、汎化性能を向上。
*   **予測精度の安定**: テストデータの正解率を **82.12%** まで改善・安定化。

### 3. 学び：データの本質を捉える重要性
*   **多角的な評価**: 正解率だけでなく、重要度スコアから「AIが納得感のある根拠で判断しているか」を検証する重要性を習得。
*   **ドメイン知識の活用**: 単体では寄与が低いデータも、背景知識（家族という集団性）に基づき統合することで、強力な指標へ昇華できることを実証。

<details>
<summary>1. 特徴量追加による精度の変動と考察 </summary>
モデル構築の過程で、以下の挙動を確認し分析を行いました。

*   **Age（年齢）**: 追加により正答率が上昇。
*   **Fare（運賃）/ SibSp（兄弟・配偶者）**: 追加により正答率が減少。訓練データに特化しすぎる**過学習（Overfitting）**の傾向を確認。
*   **重要度分析**: `Sex` > `Fare` > `Age` の順でスコアを算出。
    *   **考察**: `Fare` や `Age` が高いのは、数値が細かすぎるためにAIが「個人の特定」にデータを利用している（ノイズを学習している）状態だと判断。

### 2. 施策と改善結果

#### A. モデルの剪定（Pruning）
AIの学習範囲を制限し、細かい数値の差異ではなく「データの大きな塊」で判断するよう制御。
*   **結果**: 過学習が抑制され、未知のデータに対するスコアが向上。

#### B. 特徴量の統合
重要度スコアが低かった `SibSp` と `Parch`（親子）を統合し、`FamilySize` を生成。
*   **結果**: 正答率の直接的な劇的変化はなかったが、**重要度スコアが大幅に上昇**。データの「見せ方」を変えることで、不要に見えたデータが予測に不可欠な要素へ変わることを確認。

### 3. 使用技術・メソッド
*   **`.fillna()`**: 欠損値補完。運賃には平均値（mean）、年齢には中央値（median）を採用。
*   **特徴量エンジニアリング**: 既存データの組み合わせによる新しい意味の抽出。
